{# Backup Instructions #}
Backup

Create a database backup using the Docker CLI:

# Full database backup (MariaDB)
docker compose exec db mariadb-dump \
  -u $MARIADB_USER -p$MARIADB_PASSWORD \
  $MARIADB_DATABASE > backup_$(date +%Y%m%d).sql

Backups should be scheduled via cron or a similar scheduler for production use. Store backups in a separate location from the application server.

{# Restore Instructions #}
Restore

Restore a database from a backup file:

# Restore from backup file
docker compose exec -T db mariadb \
  -u $MARIADB_USER -p$MARIADB_PASSWORD \
  $MARIADB_DATABASE < backup_20260305.sql
Warning: Restoring a backup will overwrite all current data. Always create a fresh backup before restoring.

{# Migrations #}
Database Migrations

Manage database schema with Alembic migrations:

# Check current migration
docker compose exec web flask db current

# Apply pending migrations
docker compose exec web flask db upgrade

# Roll back last migration
docker compose exec web flask db downgrade
